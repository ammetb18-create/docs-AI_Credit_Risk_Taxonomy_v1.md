# AI Credit Governance Audit Checklist v1

This checklist provides an operational audit structure for AI-driven credit systems.

---

## 1. Model Documentation

- [ ] Model purpose formally defined
- [ ] Intended use and limitations documented
- [ ] Version control in place
- [ ] Training data sources documented
- [ ] Feature engineering logic recorded
- [ ] Model performance metrics logged
- [ ] Adverse action explanation logic validated

---

## 2. Fairness & Bias Controls

- [ ] Disparate impact testing conducted
- [ ] Protected-class proxy analysis completed
- [ ] Fairness metrics defined (e.g., demographic parity, equal opportunity)
- [ ] Thresholds for acceptable variance established
- [ ] Bias mitigation steps documented
- [ ] Periodic fairness review schedule defined

---

## 3. Regulatory Compliance Alignment

- [ ] ECOA compliance mapping completed
- [ ] Fair Lending review performed
- [ ] Audit traceability confirmed
- [ ] Decision logs retained
- [ ] Governance approval documented

---

## 4. Operational Risk Controls

- [ ] Model drift monitoring active
- [ ] Data validation checks automated
- [ ] Incident response protocol defined
- [ ] Change management documentation maintained
- [ ] Escalation pathways documented

---

## 5. Governance Oversight

- [ ] Responsible AI owner assigned
- [ ] Cross-functional oversight committee defined
- [ ] Review cadence scheduled
- [ ] Risk scoring matrix applied
- [ ] Critical risk escalation protocol defined
